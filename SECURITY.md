# Security Policy

If you discover a vulnerability in `flutter_local_agent_kit`, please do not post it in a public issue first.

## Reporting

Report security-sensitive issues through the repository owner contact listed on GitHub or by opening a private security advisory if that feature is enabled for the repository.

When possible, include:

* affected version
* platform and device details
* reproduction steps
* impact assessment
* any suggested mitigation

## Scope

Security reports are especially helpful for:

* unsafe local file handling
* unintended data exfiltration paths
* insecure model or asset download behavior
* prompt/tool execution paths that could expose user data unexpectedly

I will review reports as quickly as possible and coordinate a fix/release when needed.
